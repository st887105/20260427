1 帳號與環境準備

① Google 帳號設定

我們需要 Google 雲端硬碟來存放資料庫 (Google 試算表) 和後端應用程式 (Google Apps Script)。

關於帳號選擇：

✅ 可以使用學校教育版帳號，但需配合後續步驟的 Cloudflare Workers 來解決跨域 (CORS) 權限阻擋問題。

✅ 若無特別需求，建議使用個人的 @gmail.com 帳號，設定上最為簡便直接。

② 註冊 GitHub 帳號

GitHub 是存放前端網頁檔案並提供免費網頁代管服務 (GitHub Pages) 的平台。

  • 前往 GitHub 註冊頁面。
  • 輸入 Email，設定密碼與使用者名稱。
  • 建議：直接輸入 Email 註冊，無需使用 Google 帳號串聯，保持帳號權限獨立。
  • 至信箱收取驗證碼，完成註冊流程。

3-2 Cloudflare Workers 轉接設定 (教育帳號建議使用)

架構：前端網頁 ➔ Cloudflare Worker ➔ Google Apps Script。主要用於解決教育帳號的跨域 (CORS) 限制。

部署步驟：

  1. 登入 Cloudflare 儀表板。
  2. 左側選單點擊 Workers & Pages > Create Worker。
  3. 將第四步產出的 worker.js 程式碼貼入並儲存部署。
  4. 複製部署後取得的 *.workers.dev 網址，填入第四步表單中。

✨ 前往 EzPage 視覺化設計
💡 整合教學：如何將程式碼放進 EzPage 或上傳 GitHub？
方法 A：使用 EzPage 視覺化設計 (適合初學者)
  1. 點擊右上方「前往 EzPage」按鈕。
  2. 在 EzPage 拖拉元件設計您的專屬網頁。
  3. 將上方黑色框框內的 <script>...</script> 完整程式碼區塊複製。
  4. 在 EzPage 中加入「自訂程式碼/HTML」區塊，將複製的腳本貼上。
  5. 確保您 EzPage 中設定的按鈕點擊動作與文字框 ID，與腳本內的名稱相符。
方法 B：直接上傳 GitHub Pages (標準開發流程)
  1. 將上方程式碼直接點擊「下載檔案」(會存為 index.html)。
  2. 登入 GitHub，點擊 New repository 建立新專案。
  3. 將下載的檔案上傳至專案中。
  4. 點擊專案上方的 Settings ➔ 左側選單選 Pages ➔ Branch 選擇 main 並儲存，即可獲得公開網址！

步驟一：一鍵建立試算表欄位 (只需執行一次)
  1. 將上方代碼貼入 Google Apps Script 編輯器後儲存。
  2. 在上方工具列的「函數下拉選單」中，選擇 setupDatabase。
  3. 點擊旁邊的「執行」按鈕。
  4. ⚠ 權限審查：Google 會跳出警告視窗，依序點擊「審查權限」➔ 選擇您的帳號 ➔ 點擊左下角「進階」➔「前往 (不安全)」➔ 最後點擊「允許」。

步驟二：發布並取得 GAS 網址
  1. 點選編輯器右上角的「部署」➔「新增部署作業」。
  2. 點擊左側齒輪 ⚙️ 選擇「網頁應用程式」。
  3. 設定「執行身分」為 我，設定「誰可以存取」為 所有人 (Anyone)。
  4. 點擊「部署」，複製出現的網址。回到本系統將網址貼入上方的輸入框即可完成連動設定！
